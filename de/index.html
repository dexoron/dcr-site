DCR - Dexoron Cargo Realization

DCR ist ein Cargo-ähnliches Werkzeug für C/C++-Projekte. Die aktuelle Version ist in Rust geschrieben, unterstützt Linux, macOS und Windows und verwendet eine einzige Konfigurationsdatei: dcr.toml.

GitHub Dokumentation

Warum DCR

  • Einheitliche Projektstruktur
  • Einfache Verwaltungsbefehle
  • Unkomplizierte Einrichtung und Nutzung

Funktionen

  • Projekterstellung / Initialisierung
  • Build und Ausführung in debug/release-Profilen
  • Binär-Selbstaktualisierung über dcr --update

Plattformen

  • Linux (x86_64)
  • macOS (Intel und Apple Silicon)
  • Windows (x86_64)

Installation

Um DCR zu installieren, verwende das Terminal deines Systems:

curl -fsSL https://raw.githubusercontent.com/dexoron/dcr/master/install.sh | bash

War die Installation erfolgreich, führe folgenden Befehl zur Überprüfung aus:

                    dcr --version

Befehle

Darunterliegend sind die grundlegenden Befehle gelistet. Alle Befehle sind in der Dokumentation zu finden.

Befehl Verwendung Beschreibung
dcr dcr Hilfeinformationen anzeigen
dcr new dcr new hello Erstellt ein Projekt mit dem Namen hello im aktuellen Verzeichnis
dcr init dcr init Erstellt ein Projekt im aktuellen Verzeichnis. Der Projektname entspricht dem Verzeichnisnamen
dcr build dcr build --release Baut das Projekt. Wird kein Profil angegeben, wird --debug verwendet.
dcr run dcr run --release Baut das Projekt und führt die Binärdatei aus. Wird kein Profil angegeben, wird --debug verwendet.
dcr clean dcr clean --release Entfernt ein Build-Profil aus target. Wird kein Profil angegeben, wird der gesamte Ordner gelöscht.
dcr --update dcr --update Aktualisiert DCR auf die neueste Version aus GitHub Releases.

Beispiel

Dieser Abschnitt zeigt ein Anwendungsbeispiel für DCR.

1. Projekt erstellen

Wir erstellen ein Projekt mit dem Namen hello.

                        dcr new hello

Die resultierende Struktur sieht so aus:

                        hello/
                        - src/
                        - - main.c
                        - dcr.toml

2. Projekt bauen

Wechsle in das Projektverzeichnis:

                        cd hello

Baue das Projekt mit dem Flag --release:

                        dcr build --release

Die resultierende Struktur sieht so aus:

                        hello/
                        - src/
                        - - main.c
                        - target/
                        - - release/
                        - - - main
                        - dcr.toml
                        - dcr.lock

3. Projekt ausführen

Es gibt zwei Möglichkeiten, das Projekt auszuführen:

Erste Methode (empfohlen)

                            dcr run --release

Dieser Befehl baut das Projekt (falls nötig) und führt es aus.

Zweite Methode

                            ./target/release/main

Damit wird die Binärdatei direkt ausgeführt.

Bei Fragen zu einem der Schritte lies bitte die Dokumentation.
